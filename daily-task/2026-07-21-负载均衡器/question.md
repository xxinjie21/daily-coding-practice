## 1. 可以用几行代码实现一个负载均衡器吗？

> 📌 **大白话**：负载均衡器就像一个"分流员"，把一堆用户请求均匀分给多台服务器，别让一台累死、其他闲着。

**这道题在问什么？**
假设你开了家餐厅，雇了 3 个服务员（= 3 台服务器）。客人（= 用户请求）不断进来，你得有个规矩决定"下一个客人让谁接"。
最简单的规矩就是**轮流来**：第 1 个客人给 A，第 2 个给 B，第 3 个给 C，第 4 个又回到 A……这个"轮流"在技术上叫**轮询（Round-Robin）**。

题目问的就是：**怎么用几行 Java 代码，实现这个"轮流分配"的规矩？**

**原题给的思路（照着理解就行）**
- 准备一个**计数器**，每来一个请求就 +1，然后用 `计数器 % 服务器数量` 算出该分给哪台机器。
- 如果机器有强有弱，可以给强机器多排几个，这叫**加权轮询**：比如强机器权重 3、弱机器权重 1，就排成 `[强,强,强,弱]` 再轮流。
- 真实生产环境会更复杂：要用 Nacos / Zookeeper 自动发现机器上下线，用健康检查把坏掉的机器踢出去。

**原题里给的参考代码片段**
```java
List<String> servers = Arrays.asList("192.168.0.1", "192.168.0.2", "192.168.0.3");
AtomicInteger idx = new AtomicInteger(0);

public String getNextServer() {
    int i = idx.getAndIncrement();          // 每来一个请求，计数器 +1
    return servers.get(i % servers.size()); // 对机器数量取模，轮流选中一台
}
```
> 上面 `AtomicInteger` 你可以先理解成"一个不会出错的取号机"：很多人同时来取号，它保证每人拿到的号都不重复。
