# 题目格式说明（source-doc）

把后端面试题库以 **Markdown（.md）** 文档放进本文件夹，WorkBuddy 每日 21:00 会自动读取并随机抽一道未生成过的题。

## 约定
- 文件名随意，扩展名必须是 `.md`
- 一份文档可以含多道题；每道题用二级标题 `## ` 分隔，**标题即题目标题**
  ```markdown
  ## 如何设计一个高并发短链系统？
  原题内容……
  （建议在此写明推荐的解法 / 实现思路，自动化生成的 Demo.java 会严格按此方法来写）

  ## 如何保证 MySQL 与 Redis 双写一致性？
  原题内容……
  ```
- 若文档只有一道题、没有 `## ` 分隔，则整篇当作一道题
- **每道题建议在正文里写明推荐的解法或实现思路**：自动化生成的 `code/Demo.java` 会严格按文档里给的方法来写，不会自创另一套解法
- 题目去重依据：该题在文档里的 `## ` 标题（或首行）。已生成过的题不会重复抽
- 本说明文件本身不会被当作题目
